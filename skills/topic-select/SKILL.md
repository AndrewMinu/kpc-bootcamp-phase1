# 1단계 · 주제 선정 (topic-select)

## 목적
참가자가 **풀고 싶은 문제(이슈)** 를 하나 정한다. 지역은 아직 정하지 않는다(2단계). 정답을 주지 않고 참가자가 고르게 한다.

## 시작 전
- `context.md`가 없으면 `bootcamp-guide/references/_context-template.md`를 현재 폴더에 복사해 만든다.
- 주제카드: 스킬과 함께 설치된 **형제 폴더 `bootcamp-guide/references/cards/`**(카드_*.md, README.md). 없으면 프로젝트 `ft-skills/pipeline/cards/`. → 어느 폴더에서 켜도 동작.

## 참가자에게 보여줄 범위 (중요)
주제카드에서 **1~4번만** 보여준다:
1. 문제 중심 주제(이슈) · 2. 대상 사용자/페르소나 · 3. 검증 전 가설 · 4. ★후보 지역 선택표
> 5(단서)·6(아웃풋)·7(MVP)·확장 레이어는 **이 단계에서 숨긴다**(뒤 단계·FT용). 후보 지역표는 "이런 지역들이 있다" 참고용으로만 보여주고, 실제 지역 선택은 2단계에서 한다.

## 진행 (퍼실리테이션)
0. **시작 확인 — 이미 정했나, 같이 고를까.** 먼저 묻는다: *"하고 싶은 주제가 이미 있어요? 아니면 같이 골라볼까요?"*
   - **이미 있으면**: 그 주제를 듣고, 20개 카드 중 가까운 게 있으면 매칭해 보여준다(맞으면 그걸로, 살짝 다르면 자유주제 경로). 억지로 카드에 끼워맞추지 않는다.
   - **같이 고르고 싶으면**: 아래 1~3으로 퍼실리테이션.
1. **팀 생각 먼저(개방형).** 카드를 쏟기 전에: *"우리 지역·관광에서 평소 뭐가 문제 같았어요? 각자 한 가지씩 말해볼래요?"* 팀원들이 관심·경험을 먼저 꺼내게 한다(한 명 독점 금지). 잘 안 나오면 그때 분야 예시(체류·숙박 / 콘텐츠·발견 / 교통 / 축제 / 로컬브랜드 / 트렌드 / 저활용)를 **보조로**.
2. **팀 관심과 카드 잇기** — 팀이 말한 것과 겹치는 이슈 몇 개를 `[키] 제목` 한 줄로 보여주고 *"이 중에 우리가 얘기한 거랑 가까운 게 있어요?"*. 원하면 전체 20개도 열어준다.
3. **선택 카드 상세** — 팀이 끌린 카드의 1~4번만 펼쳐 보여주고 *"이 문제가 우리가 하고 싶은 거랑 맞아요? 어디가 공감되고 어디가 아쉬워요?"* 로 팀 토의를 유도. 합의로 정한다.
4. **경로 분기**
   - **카드 선택**: `경로=카드`, `주제=[키] 제목`.
   - **자유주제(카드 외)**: "카드에 없는 걸 하고 싶다"면 직접 입력받는다 — 문제 한 줄 + 대상 사용자 + 왜 문제인지. `경로=자유주제`, `주제=자유주제 텍스트`. ⚠자유주제는 **2단계 지역 선택도 자유이고, 4단계 데이터 검증이 필수**임을 알린다(가설이 데이터로 서는지 반드시 확인).
5. **context.md 갱신** — `경로`·`주제`·`단계: 1` 기록, 로그 추가.

## 끝맺음
- 요약: "주제 = ○○ (경로: 카드/자유주제)".
- **다음: `region-context` 스킬** — 후보 지역 중 1곳을 골라 컨텍스트를 잡습니다. (자유주제면: 원하는 지역을 정하고 데이터로 검증합니다.)

## 톤
- 초보 기준, 한 번에 과하게 쏟지 않기. "정답은 없다 — 관심과 근거로 고른다."
- 카드 문구의 '~일 수 있다' 가설형을 그대로 유지(단정 금지).

# 데이터 카탈로그 (다운로드 링크 & 분석 방법)

> `tourism-data-validator` 스킬과 참가자 실습이 공통으로 참조하는 데이터 출처 목록.
> 링크가 바뀌면 이 파일만 갱신한다. (확인일: 2026-06)

## 핵심 원칙 (데이터 소스 티어)
- **기본(필수)**: **한국관광 데이터랩** CSV/Excel 다운로드 = *수요·소비 신호*. 모든 카드 최소 1개 사용.
- **심화(선택)**: **공공데이터포털**, **통계청 SGIS**, **NIA AI 허브(문화관광 탭)** = *공급·맥락 보강 / 추가 과제*.
- **통합 키**: 국내 **공공데이터포털 API 키 1개**로 TourAPI·문화공연 등 대부분 공공 API 통합 접근.
- 분석은 평균·합계·순위·비교·증감만. (→ `bootcamp-context.md` §3)

## 지역별 TourAPI 공급량 레퍼런스 (시드 5개, 라이브 검증값 2026-06-24)

> ⚠ **시드 5개 = 검산용 앵커**일 뿐, 스코프가 아니다. 실제 1번 수집은 **전국 시군구 전수**가 목표(공급은 `areaBasedList2`로 자동 수집 가능). 이 표는 형식 참고 + 결과 검산용.
> TourAPI `areaBasedList2` totalCount 실측. (수치는 데이터 갱신으로 변동 → 작성 시점에 재호출. 검증 상세: `validation/card-supply-validation-2026-06-24.md`)

|---|---|---|---|---|---|---|---|
### ⚠ 데이터 특성 주의 (안정 vs 변동)
- **안정(영구 시설)**: 관광지·문화·숙박·음식·레포츠·쇼핑 = 실제 장소라 값이 안정적. 수집값=라이브 일치(검증됨). 시그니처에 신뢰 사용.
- **변동(이벤트)**: **축제(festival)** = TourAPI가 **끝난 축제를 삭제**하므로 "현재 등록된 축제"만 반환 → 계절·시점에 따라 값이 크게 변함(예: 목포 6월 3 → 7월 0). `searchFestival2`로 과거 날짜를 넣어도 이미 종료된 축제는 안 나와 **안정적 연간 축제 수는 TourAPI로 불가**.
  - 대안: 축제 상시 지표가 필요하면 **문화관광축제 지정 목록** 또는 **문화체육관광부 문화예술공연 통합 API**(연간). 또는 축제는 *부트캠프 시기 기준 스냅샷*으로만 사용.

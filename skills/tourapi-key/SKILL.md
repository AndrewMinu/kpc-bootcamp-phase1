# 3단계 · TourAPI 키 발급 (tourapi-key)

## 목적
참가자가 **자기 공공데이터포털 인증키 1개**를 발급받아 환경변수로 등록한다. 이 키로 4단계에서 실시간 데이터를 받는다.

## 보안 관점 (짚고 가기)
- 이 키는 **공공데이터포털 무료 인증키**로 민감도가 낮다(요금 없음·요청 제한 있음·언제든 재발급/폐기 가능). 비밀번호·결제정보가 아니다.
- 그래도 **깃 커밋·카드·`context.md`에는 저장하지 않는다.** 회원가입·발급은 참가자 본인이 직접.
- 자세한 원본: `bootcamp-guide/references/guides/api-setup.md`(없으면 `ft-skills/shared-context/api-setup.md`).

## 발급·신청 (참가자 본인)
1. [공공데이터포털](https://www.data.go.kr) 회원가입·로그인.
2. 아래 API에서 각각 **[활용신청]** (대부분 즉시 자동승인·무료):
   - 한국관광공사_국문 관광정보 서비스_GW (TourAPI, 장소 공급) — data.go.kr/data/15101578/openapi.do
   - 한국관광공사_빅데이터_지역별 방문자수 (수요) — data.go.kr/data/15101972/openapi.do
   - (선택) 관광지별 연관 관광지 정보 — data.go.kr/data/15128560/openapi.do
3. **마이페이지 > 인증키 발급 현황**에서 **일반 인증키(디코딩)** 1개 확인. 이 키 하나로 위 API 전부 사용.

## 키를 스킬이 쓰게 하는 법 (택1) — Codex 주의 포함
> ⚠ **Codex 함정**: 다른 터미널 창에서 `export` 한 환경변수는 **Codex가 명령을 실행하는 셸 세션에는 안 보일 수 있다**(별도 셸). 그래서 아래 방법을 쓴다.

**A) 프로젝트 `.env` (권장·재입력 불필요)**
프로젝트 폴더에 `.env` 파일을 만들고:
```
DATA_GO_KR_SERVICE_KEY=발급받은_일반_인증키_Decoding
```
그리고 `.gitignore`에 `.env` 추가(커밋 방지). 스킬/스크립트는 호출 전에 `.env`를 읽어 키를 쓴다. 세션 내내 유지된다.

**B) 채팅에 붙여넣기 (빠름)**
참가자가 키를 채팅에 붙여넣으면 스킬이 **그 세션 호출에만** 사용한다. 파일·`context.md`에 저장하지 않는다.
- 저민감 키라 실무상 무난하지만, **대화 로그엔 남는다** → 테스트가 끝나면 원하면 data.go.kr에서 키 재발급으로 폐기 가능.
- 붙여넣은 값을 화면에 다시 출력하지 않는다.

> ⚠ **신규 키는 즉시 안 된다.** 발급/승인 직후 최대 1시간 게이트웨이 반영 지연. 이때 호출하면 XML 에러가 아니라 평문 `Unauthorized`가 온다 = 아직 미반영. 잠시 후 재시도.

## 활성화 확인 (스킬이 수행)
키를 확보하면(`.env` 또는 채팅) 테스트 호출로 활성 여부만 확인한다(값은 출력하지 않는다):
```
https://apis.data.go.kr/B551011/KorService2/areaBasedList2?serviceKey={KEY}&numOfRows=1&pageNo=1&MobileOS=ETC&MobileApp=ftskill&arrange=A&contentTypeId=12&areaCode=1&sigunguCode=1&_type=json
```
- `resultCode:0000` → **활성**. `Unauthorized`/`Forbidden` → 미반영/미신청 → 안내 후 잠시 뒤 재시도.

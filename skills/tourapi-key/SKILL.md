# 3단계 · TourAPI 키 발급 (tourapi-key)

## 목적
참가자가 **자기 공공데이터포털 인증키 1개**를 발급받아, **채팅에 붙여넣기만** 하면 4단계에서 실시간 데이터를 받을 수 있게 한다. (환경변수·설정 몰라도 됨)

## 말투
가볍고 친절하게. 처음 해보는 사람이 많으니 겁주지 말고 "생각보다 금방 돼요 🙂" 톤으로 한 단계씩.

## 보안 관점 (짚고 가기)
- 이 키는 **공공데이터포털 무료 인증키**로 민감도가 낮다(요금 없음·요청 제한 있음·언제든 재발급/폐기 가능). 비밀번호·결제정보가 아니다.
- 그래도 **깃 커밋·카드·`context.md`에는 저장하지 않는다.** 회원가입·발급은 참가자 본인이 직접.
- 자세한 원본: `bootcamp-start/references/guides/api-setup.md`(없으면 `ft-skills/shared-context/api-setup.md`).

## 발급·신청 (참가자 본인)
1. [공공데이터포털](https://www.data.go.kr) 회원가입·로그인.
2. 아래 API에서 각각 **[활용신청]** (대부분 즉시 자동승인·무료):
   - 한국관광공사_국문 관광정보 서비스_GW (TourAPI, 장소 공급) — data.go.kr/data/15101578/openapi.do
   - 한국관광공사_빅데이터_지역별 방문자수 (수요) — data.go.kr/data/15101972/openapi.do
   - (선택) 관광지별 연관 관광지 정보 — data.go.kr/data/15128560/openapi.do
3. **마이페이지 > 인증키 발급 현황**에서 **일반 인증키(디코딩)** 1개 확인. 이 키 하나로 위 API 전부 사용.

## 키를 스킬에 넘기는 법 — 그냥 채팅에 붙여넣기 (기본)
참가자가 **키를 채팅에 그대로 붙여넣으면** 스킬이 알아서 씁니다. 환경변수·`.env` 설정 몰라도 됩니다.
- *"발급받은 키를 여기 채팅에 붙여넣어 주세요. 그럼 제가 이번 세션 동안 데이터 받을 때 씁니다."*
- 이 키는 **무료·저민감 공공데이터 키**라 이렇게 해도 실무상 무난. 다만 **파일·`context.md`·git엔 저장하지 않고**, 화면에 값을 다시 출력하지도 않는다. (학습자는 어차피 git 커밋 안 하니 그대로 OK)
- 세션이 바뀌어 키를 또 물으면 다시 붙여넣게 하면 된다.

> (선택·앱 제출자용) 별도 앱을 만들어 제출할 팀은 키를 코드에 박지 말고 프로젝트 `.env`(`DATA_GO_KR_SERVICE_KEY=...`, `.gitignore`에 추가)에 두는 습관을 들이면 좋다. ⚠ 참고: 다른 터미널에서 `export` 한 값은 Codex 셸에 안 보일 수 있으니 `.env`를 쓴다.

> ⚠ **신규 키는 즉시 안 된다.** 발급/승인 직후 최대 1시간 게이트웨이 반영 지연. 이때 호출하면 XML 에러가 아니라 평문 `Unauthorized`가 온다 = 아직 미반영. 잠시 후 재시도.

## 활성화 확인 (스킬이 수행)
키를 받으면 테스트 호출로 활성 여부만 확인한다(값은 출력하지 않는다):
```
https://apis.data.go.kr/B551011/KorService2/areaBasedList2?serviceKey={KEY}&numOfRows=1&pageNo=1&MobileOS=ETC&MobileApp=ftskill&arrange=A&contentTypeId=12&areaCode=1&sigunguCode=1&_type=json
```
- `resultCode:0000` → **활성**. `Unauthorized`/`Forbidden` → 미반영/미신청 → 안내 후 잠시 뒤 재시도.

## context.md 갱신
- `TourAPI키: 등록됨`(.env/채팅 확보) 또는 `활성확인`(테스트 통과). **키 값은 절대 기록하지 않는다.**
- `단계: 3`, 로그 추가.

- **다음(데이터) 단계 스킬이 `/skills`에 보이면** `$data-collect-validate`로 이어가고, **아직 안 열렸으면** 다음 단계가 열릴 때까지 재촉하지 말고 팀과 아래를 더 나눈다:
  - *"우리 주제·지역, 팀원 모두 진짜 납득했어요? 갸우뚱한 사람 없어요?"*
  - *"이 지역에서 왜 그 문제가 생겼을지, 우리 나름의 가설을 더 세워볼까요?"*
  - *"우리 사용자가 누굴지 미리 한 명만 그려볼까요? 다음 단계에서 크게 쓰여요."*
  - 시간 되면 `$field-research`로 그 지역·사용자 이야기를 미리 들어봐도 좋다.
- **키가 아직 미반영(Unauthorized)**이면: 1시간쯤 뒤 다시 확인하면 된다고 안내하고, 그동안 위 확장 질문으로 대화를 이어간다.
